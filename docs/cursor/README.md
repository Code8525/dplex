# Правила Cursor для проекта dplex

Описание того, как устроены правила Cursor и какое правило для dplex подключено.

## Как устроены правила Cursor

- Правила лежат в **`.cursor/rules/`** в виде файлов **`.mdc`**.
- В начале файла — **YAML frontmatter**:
  - `description` — краткое описание (показывается в выборе правил);
  - `globs` — шаблон файлов, при открытии которых правило активно (например `**/*.py`);
  - `alwaysApply` — если `true`, правило действует в каждом чате.
- Тело правила — Markdown с указаниями и примерами кода.

Подробнее: [Cursor Rules](https://docs.cursor.com/context/rules-for-ai), skill `create-rule`.

## Подключённое правило для dplex

Файл **`.cursor/rules/dplex-sort-filters.mdc`** задаёт:

1. **Опциональный sort_by**  
   При типовом API-запросе списка с полями `sort_by` и `sort`:
   - использовать один элемент: `Sort(by=request.sort_by, order=Order(request.sort))`;
   - не подставлять поле по умолчанию при отсутствии `sort_by`;
   - не заменять выбор клиента жёстким набором полей сортировки.

2. **DPFilters и Sort**  
   Использование `DPFilters[SortFieldEnum]`, поля `sort`, допустимость `Sort(by=None)`, множественная сортировка списком `Sort`, поведение `has_sort()`.

Правило применяется при работе с файлами `**/*.py`. При необходимости измените в frontmatter `globs` (например, только API: `**/api_backoffice/**/*.py`) или установите `alwaysApply: true`.

## Как добавить или изменить правило

1. Создайте или отредактируйте файл в `.cursor/rules/*.mdc`.
2. Заполните frontmatter и текст правила (лаконично, с примерами).
3. Рекомендуется держать правило в пределах ~50 строк и одной темы.

Пример минимального frontmatter для правила только для Python:

```yaml
---
description: Краткое описание
globs: **/*.py
alwaysApply: false
---
```
